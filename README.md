### 株式会社グロービー（2021年3月 〜 現在）

#### AI英語学習サービス「abceed」のWebフロントエンド開発

**期間：** 2021年3月 〜 現在
**役割：** フロントエンドリード / チームリーダー

**概要：**
AI英語学習サービス「abceed」のWebフロントエンド開発をリード。入社時点でWebフロントエンドエンジニアは自分を含め2名（現在4名）という少人数体制の中、abceed本体や管理画面の開発を担当。チームリーダーとして採用（コーディング試験の作成・技術面談）、タスク・リソース管理、プロジェクト進行管理も担っている。

**課題と取り組み：**

- 映画・ドラマを使った英語学習機能の新規開発で、DRM対応の動画プレイヤー実装が必要だった
  - hls.jsでプロトタイプを素早く構築し、映画会社とのUI/UX検証を実施
  - DRM（Widevine）対応でVideo.jsに移行したが、フレーズ単位の高速シークで動画が停止する問題が発生
  - 最終的にShaka Playerへ移行し、DRM対応とフレーズ単位のシームレスな再生を両立
  - 詳しくは[abceed webの映画・ドラマ機能開発で使用した動画ライブラリの変遷](https://qiita.com/umi_kappa/items/f91eb2ed1ea3e0594992)をご覧ください
- Vue 2で構築された既存コードベースに技術的負債が蓄積していた
  - 新規機能開発と並行しながらVue 3への段階的な移行を推進中
- デザインチームが作成したデザインを操作感・実装観点でレビューし、仕様調整を積極的に行った
- Gitの運用ルールの見直しやリリースフローの改善に取り組んだ

**AI活用による開発効率化（現在も積極的に検証・導入中）：**

- Cursor、CodeRabbitによるコーディング・PRレビューの効率化
- Claude Codeを活用したテスト自動生成、Figma MCPによるスタイリング
- Git Worktreeでの並行開発、PRレビューエージェントやリリースIssue作成スキルを自作し、リーダー業務と開発の両立を改善

**技術スタック：** TypeScript, Vue.js (Vuex / Pinia), Vite, webpack, Vitest, Storybook, GitHub Actions, AWS, Figma

#### 車両遠隔操作システムのWebアプリケーション開発

**期間：** 2019年1月 〜 2019年3月
**役割：** フロントエンド開発

**概要：**
国内自動車メーカー向けに、ラジコンカーをスマートフォン（Webアプリ）から遠隔操作するデモ用アプリケーションを開発。営業担当者がお客様の前でテンポよく操作できることが求められた。

**課題と取り組み：**

- ジョイスティックUIにHammer.jsを採用したが、操作感（レスポンスの悪さ）やマルチタップの面で課題があった
  - タッチ操作に最適化したジョイスティックUIをスクラッチで開発し、直感的な操作性を実現
- デザイナーと協業し、失敗してもフォールバックできるフローなど細部のUXにこだわった
  - 納品時にクライアントが説明なしで操作できるUXを達成

**技術スタック：** TypeScript, React, Redux, Jest, Storybook

#### 車載器ステータス確認用Webアプリケーション開発（Bluetooth）

**期間：** 2018年9月 〜 2018年12月
**役割：** フロントエンド開発（設計〜実装）

**概要：**
自社製車載器のステータス情報をスマートフォンで確認するWebアプリケーションを開発。インターネット接続がない環境でも動作するよう、Web Bluetooth APIを活用したオフライン対応を実現。

**課題と取り組み：**

- ネットワーク接続がない環境でも動作する要件があった
  - ServiceWorker（Workbox）によるローカルキャッシュとWeb App Manifestでオフライン起動を実現
  - Web Bluetooth（BLE）でハードウェアとの無線通信（Read・Notification）を実装
- Web Bluetooth APIの事例情報が極めて少なく、仕様も策定途中だった
  - BLE仕様を読み込み、未知のAPIを一から検証しながら機能要件を満たす実装を完成させた

**技術スタック：** TypeScript, React, Redux, ServiceWorker, Storybook
